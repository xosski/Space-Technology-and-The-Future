1. Core Concept

REZ Lazarus is a top-and-bottom Lazarus capacitor array mounted around a ship’s central protected spine. The system stores high-density energy in paired capacitor banks and releases it in rapid, controlled bursts.

The goal is not normal propulsion.

The goal is:

store hard
release fast
shape outward
survive the backlash

In your diagram:

Red diamonds = REZ Lazarus capacitor/discharge arrays
Yellow rectangle = outer ship hull / armored energy bus
Blue rectangle = protected core / internal power spine

The top and bottom arrays create a biaxial defensive discharge system. When fired together, they push energy away from the hull in opposite directions, creating a temporary protective bloom, flare, or dispersal field.

The Prismatic Lazarus Capacitor already defines this role well: a rapid pulse reservoir used for shield recharge, plasma pulse, emergency power dump, and short-duration power surges.

GhostCore phrasing:

The ship does not hold lightning in one fist.
It places a storm above,
a storm below,
and dares impact to choose poorly.
2. Main Hypothesis

If high-capacity Lazarus capacitor banks are mounted above and below the ship’s central core, then synchronized discharge can create a balanced defensive energy dispersal effect.

The system can be used to:

- reinforce shields
- vent overload away from the hull
- disperse incoming plasma or EM energy
- break target lock with burst interference
- create a temporary defensive pulse shell
- feed emergency evasive systems
- absorb and redirect surge through the Elohim Reserve Lattice

The system does not create free energy.
It does not make the ship invincible.
It does not bypass thermal or structural limits.

It stores energy and releases it violently, but with manners. Ideally.

- spread energy across the hull surface
- convert raw discharge into defensive field geometry

Without this, REZ is just a capacitor yelling at the universe.

The Elohim Reserve Lattice is the shipwide reserve and surge-management layer.

Its job:

- feed charge into REZ arrays
- absorb discharge backlash
- diffuse overload
- reroute emergency power
- prevent one capacitor bank from becoming a single catastrophic failure point

The Elohim Reserve Lattice is designed specifically for distributed storage, surge absorption, controlled dissipation, and emergency power rerouting.

D. Discharge Gates

The discharge gates control how stored energy leaves the arrays.

Possible output forms:

- electric pulse
- EM burst
- plasma flare
- shield recharge pulse
- overload vent
- sensor-blinding interference flare

The gate is the leash.

Without the gate, the capacitor is not a system. It is a future accident.

E. Control Core

The control core synchronizes the top and bottom arrays.

It monitors:

- capacitor charge
- hull field stress
- thermal load
- top/bottom balance
- incoming threat vector
- lattice absorption capacity
- discharge-gate timing

Its primary duty is to prevent the arrays from firing unevenly unless asymmetric discharge is deliberately requested.

8. Control Logic
def rez_lazarus_step(ship, rez_top, rez_bottom, lattice, threat, command):
    """
    Speculative control step for the REZ Lazarus defensive dispersal array.
    """

    # 1. Read system state
    top_charge = rez_top.available_energy
    bottom_charge = rez_bottom.available_energy
    lattice_margin = lattice.available_absorption
    hull_stress = ship.hull_field_stress
    thermal_margin = min(rez_top.thermal_margin, rez_bottom.thermal_margin)
    threat_vector = threat.vector

    # 2. Safety checks
    if thermal_margin < 0.2:
        return emergency_bleedoff(
            rez_top, rez_bottom, lattice,
            reason="thermal_margin_low"
        )

    if lattice_margin < command.minimum_lattice_margin:
        return hold_fire(
            rez_top, rez_bottom,
            reason="insufficient_lattice_absorption"
        )

    if abs(top_charge - bottom_charge) > command.max_charge_imbalance:
        return rebalance_arrays(
            rez_top, rez_bottom, lattice,
            reason="top_bottom_charge_mismatch"
        )

    # 3. Select operating mode
    if command.mode == "defensive_bloom":
        pulse_plan = synchronize_biaxial_pulse(
            top=rez_top,
            bottom=rez_bottom,
            intensity=command.intensity,
            phase="matched",
        )

    elif command.mode == "dispersal_curtain":
        pulse_plan = staggered_pulse_train(
            top=rez_top,
            bottom=rez_bottom,
            interval_ms=command.interval_ms,
            cycles=command.cycles,
        )

    elif command.mode == "overload_vent":
        pulse_plan = outward_dump_sequence(
            top=rez_top,
            bottom=rez_bottom,
            dump_limit=command.dump_limit,
        )

    elif command.mode == "snap_roll":
        pulse_plan = asymmetric_snap_pulse(
            top=rez_top,
            bottom=rez_bottom,
            offset_ms=command.offset_ms,
            torque_limit=ship.max_safe_torque,
        )

    else:
        return standby_status(rez_top, rez_bottom, lattice)

    # 4. Fire discharge gates
    rez_top.discharge(pulse_plan.top)
    rez_bottom.discharge(pulse_plan.bottom)

    # 5. Route backlash into Elohim lattice
    lattice.absorb_transient_load(
        electrical_spike=pulse_plan.estimated_backlash,
        thermal_spike=pulse_plan.estimated_heat,
        field_echo=pulse_plan.estimated_field_echo,
    )

    # 6. Update ship state estimate
    ship.hull_field_stress += pulse_plan.estimated_hull_stress
    ship.thermal_load += pulse_plan.estimated_heat

    return {
        "mode": command.mode,
        "top_charge": rez_top.available_energy,
        "bottom_charge": rez_bottom.available_energy,
        "lattice_margin": lattice.available_absorption,
        "status": "pulse_executed",
    }
