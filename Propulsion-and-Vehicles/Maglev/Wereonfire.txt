Proof-of-Concept: Resonant Field “Snap Rail” Mobility System (A + B Add-On)
High-level concept

A guided corridor (rail) with two opposing field sources (“PTECs”) that can be tuned so the system shifts between two stable field configurations:

State HOLD: stable capture / centering (minimizes drift)

State DRIVE: stable forward bias (creates net pull/push along axis)

That state transition is your “snap”: not snapping physics itself, but snapping the equilibrium mode.

POC A — Guided Hover + Transport Rail
A1) Goal

Demonstrate a carriage that:

maintains a non-contact standoff (or near-contact ultra-low friction if you choose non-superconducting route),

stays centered in the corridor, and

moves along the rail under controlled field actuation.

A2) Two build routes (pick what you can actually source)
Route A1 (Fastest, proven): Superconductor hover demo rail

Uses YBCO puck + liquid nitrogen for hover/locking.

Track is a magnet array.

“Car” is a platform carrying the puck.

Why it’s ideal: it immediately proves hover + stability without fighting control theory.

Route A2 (No cryo): Electromagnet “active suspension” rail

Uses electromagnets + sensors for standoff control.

More engineering, less spectacular, still valid.

Why it’s practical: no LN₂ requirement.

If you want my recommendation: A1 first if you can access LN₂, otherwise A2.

A3) Hardware overview (A1 hover rail)
Components

Superconductor puck (YBCO)

LN₂ container + safety gear

Magnet track (two side rails + optional center rail)

Guide walls (keeps it from ejecting)

Carriage platform (lightweight)

Track geometry (stable “corridor”)

Build a “magnetic trough”:

Left rail magnets oriented to create a lateral restoring force

Right rail mirrors it

Optional center strip for extra lift/locking

Acceptance tests (POC A pass/fail)

Hover lock time: ≥ 60 seconds stable on track

Centering: stays within corridor without touching sidewalls for ≥ 30 seconds

Travel: moves ≥ 0.5 m end-to-end (pushed by hand is OK for A-phase; B will motorize)

POC B — Add-On 1: “Snap” Field-State Switching (Bistable Mode Demo)

This is the part that answers your question about “snapping EM fields” using paired PTECs and a reverse rail with different polarity/bias.

B1) Goal

Show that the corridor has two distinct, repeatable equilibrium states and you can switch between them via controlled parameters:

relative phase (Δφ) or

amplitude ratio (A₁/A₂) or

bias sign (a “polarity” flip in one rail)

B2) What you’re proving (clean definition)

A “snap event” is defined as:

When you slowly sweep a control variable, the system stays in State A until a threshold, then rapidly transitions to State B, and remains there even if you slightly reverse the sweep (hysteresis).

That hysteresis loop is the signature of bistability.

B3) Buildable lab implementation (safe + measurable)
The corridor sources (your “PTECs” as resonant field modules)

For a POC you don’t need exotic hardware — you need two coupled field emitters you can tune:

Coil Module 1 (Top rail): a driven coil/loop/solenoid assembly

Coil Module 2 (Bottom rail): a second driven coil/loop/solenoid assembly

Both driven from signal sources where you can control:

amplitude (A)

phase (φ)

frequency (f)

DC bias (optional)

Practical note: In POC land, “PTEC” = “tunable resonant field module.” The geometry can evolve later.

The payload/coupler (what responds)

Pick one:

conductive ring / aluminum loop (eddy-current response)

small magnet + spring mount (mechanical indicator)

Hall sensor sled (field mapping)

Sensors (minimum viable)

2–4 Hall sensors along the corridor (or one on a moving sled)

Optional: a photogate or IMU if something actually moves

B4) Snap test procedure

Set frequency (f) to your chosen operating point.

Set A₁ = A₂ (balanced).

Sweep Δφ slowly from 0 → 180° while logging Hall sensor values.

Observe:

Does the field pattern shift smoothly?

Or does it “jump” at some Δφ* threshold?

Repeat while sweeping back (180° → 0) to see if the threshold differs (hysteresis).

Repeat using bias sign (flip rail 2 DC bias) instead of phase.

B5) “Snap” success criteria

A repeatable transition point within ±5–10% of sweep range

Distinct sensor signatures for State A vs State B

Optional: visible “latching” behavior of a coupler (it wants one position, then abruptly wants another)

POC B — Add-On 2: Plasma Thermal Module (Cooling/Heat-Flow Experiment)

This is the “plasma supercooling” theme translated into a measurable thermal control test.

C1) Goal

Demonstrate that a controllable plasma discharge (commercial module) measurably changes heat transfer from a target.

C2) Hardware

A small heater plate (aluminum/copper) with a resistive heater pad

Temperature sensor (thermistor/RTD/thermocouple)

Data logger (Arduino/ESP32 is fine)

A commercial cold plasma module (enclosed, rated) for airflow/discharge across the target

C3) Procedure

Apply constant heater power until temperature stabilizes.

Turn plasma ON; log temperature vs time.

Turn plasma OFF; log return.

Repeat 3–5 times.

C4) Success criteria

A consistent change in equilibrium temperature (ΔT) or cooling time constant (τ)

Repeatable effect across multiple runs
